# Changelog NeoPSE - v0.2.0 "Modular Engine"

Hito fundamental en la arquitectura del proyecto: Transición de un sistema monolítico a un motor desacoplado basado en Registros y Sistemas de Input globales.

## 🚀 Nuevas Capacidades
- **Block Registry:** Sistema data-driven para registrar nuevos tipos de bloques sin tocar el código core.
- **Input System:** Orquestador global de teclado. Soporte inicial para `Esc` (cancelar acciones).
- **Auto-Persistencia (Rehydration):** El workspace ahora recuerda bloques, posiciones y enlaces entre sesiones.
- **Geometry Utils:** Librería matemática unificada para cálculos de distancia e intersecciones.
- **Space Manager:** Fuente de verdad única para transformaciones de coordenadas en el canvas infinito.

## 🛠️ Refactorización y SOLID
- **Herencia POO:** Implementación de `EditorBlock`, `NoteBlock` y `AssistantBlock` heredando de una base abstracta.
- **Event Delegation:** Los bloques ya no tienen listeners individuales; usan delegación de eventos para las acciones (Link, Close).
- **Selective Redraw:** El motor de enlaces ahora solo actualiza los cables afectados por el movimiento, optimizando el rendimiento en mapas grandes.
- **CSS Design System:** Todas las dimensiones y jerarquías visuales ahora dependen de variables CSS centralizadas.

## 🛡️ Blindaje Técnico
- Eliminación de "Magic Numbers" en todo el código.
- Centralización de tipos y configuraciones en `Constants.ts`.
- Sincronización matemática total entre el motor de arrastre y el de dibujo.
