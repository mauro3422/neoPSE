# Changelog NeoPSE - v0.2.5 "Interface Hardening"

Refactorización profunda de la base de componentes para unificar la lógica de interfaz fija y mundo dinámico.

## 🚀 Mejoras de Arquitectura
- **UIComponent Unificado:** Nueva clase base que maneja la persistencia de forma automática tanto para paneles fijos (Asistente) como para bloques móviles.
- **Persistencia Transparente:** Los métodos `getStateData()` y `syncState()` ahora están centralizados, eliminando código duplicado en todos los componentes especializados.
- **Preparación para Módulos:** La arquitectura ahora es compatible con la inyección de nuevos paneles (ej. Árbol de Archivos, Consola) sin tocar el código core.

## 🛠️ Refactorización DRY & SOLID
- **AssistantBlock:** Limpieza total de lógica redundante. Ahora usa el motor de persistencia estándar de la aplicación.
- **SpecializedBlocks:** Reducción de código en un 40% al delegar la gestión de eventos e inputs a la clase base `Block`.
- **Harden de Eventos:** El sistema de "Auto-Save" ahora es una capacidad intrínseca de cualquier componente que herede de `UIComponent`.

## 🛡️ Estabilidad
- Corregidos conflictos de posicionamiento entre capas HUD y WORLD.
- Mejorada la rehidratación de contenidos JSON (Chat) para evitar corrupciones de datos.
